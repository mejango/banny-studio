# v1.3.0 — Built-in Backdrops, Square Frame, Auto-Frame — Design

**Date:** 2026-07-21
**Status:** Approved

## Goal

Ship v1.3.0 to the Mac App Store with: (1) the 53 reference GIFs bundled as
built-in backdrop options, (2) a Square (1:1) frame preset, (3) frame size
that auto-adjusts to the first backdrop when the user hasn't chosen a frame.
The release also folds in the pending export-preferences / track-transfer
working-tree WIP.

## 1. Built-in backdrop gallery

- Source: the 53 `.gif` files from `/Users/jango/Downloads/GIf`
  (`teach-frame.png` excluded — it is a frame overlay, not a backdrop).
  All ~1200×1200 (one 1200×1390); ~3.6 MB total.
- They ship in the app bundle at `App/Resources/Backdrops/` (added to
  `App/project.yml` resources).
- Display names derive from filenames: strip extension, `-`/`_` → space,
  capitalize words; a trailing "no banny" becomes "(no banny)".
- UI: the Assets section in `App/Sources/SidePanelSections.swift` gains a
  "Built-in backdrops" browser above the imported-assets list: scrollable
  thumbnail grid (first GIF frame), name on hover/caption. Clicking one:
  1. copies the GIF bytes into the document's asset bank
     (`Asset(kind: .image, file: <name>.gif)` + package `assets/`), reusing
     the existing imported-asset code path;
  2. adds a background cue via the existing `addBackgroundCue`;
  3. triggers the auto-frame rule below.
- Projects stay self-contained: shared `.bs` files and `banny ship` render
  bundled backdrops without the app present.
- Adding the same backdrop twice reuses the already-imported asset (match
  by source filename) instead of duplicating bytes.

## 2. Square frame preset

- `FrameSection` presets: 16:9, 9:16, **Square (1:1)**, Custom.
- Custom's sentinel changes from `setFrame(1, 1)` to `setFrame(4, 3)`
  (1:1 now belongs to Square). Preset detection: 16/9 → horizontal,
  9/16 → vertical, equal W/H → square, else custom.
- No exporter change needed: `Options.fitted(aspect:)` already reshapes
  (Square 1080p tier → 1080×1080).

## 3. Auto-frame rule

When a background cue is added (gallery click or user-imported asset):
adjust `settings.frameW/frameH` to the asset's pixel aspect IFF

- the document currently has **zero** background cues, AND
- the frame is the untouched default (`frameW == 16 && frameH == 9`).

The aspect is stored as the reduced integer ratio of the asset's pixel
size (1200×1200 → 1/1; 1200×1390 → 120/139). The change registers an undo
snapshot ("Frame"). Decision logic is a pure function in BannyCore
(`Settings.autoFrame(assetW:assetH:hasBackgroundCues:) -> (Double, Double)?`)
with unit tests; `StudioModel` applies it.

## 4. Release v1.3.0 (macOS App Store)

- First: commit the working-tree WIP (export preferences, track transfer,
  ShowExporter bitrate retunes) after a full test pass — it is release
  content ("the export preferences update").
- iOS compile fix only: guard the `Process`/ditto usage in
  `App/Sources/ShowDocumentFile.swift` (and any other `Process` use in app
  code) behind `#if os(macOS)` so the iOS destination builds again. iOS
  `.bs` zip import is out of scope (needs a zip library) — **this
  submission is macOS-only**; iOS ships later.
- Version: marketing version 1.3.0 in `App/project.yml` (+ build number
  bump), `xcodegen`, archive macOS, export with App Store signing, attempt
  `xcrun altool`/notarytool upload. If CLI auth is unavailable, stop and
  hand the user the Organizer step.

## Out of scope

- iOS App Store submission (blocked on zip import approach).
- Backdrop search/categories in the gallery; curation.
- CLI awareness of bundled backdrops (projects embed the asset, so the CLI
  needs nothing new).

## Testing

- BannyCore: unit tests for `Settings.autoFrame` (fires only on default
  frame + no cues; reduced ratios; nil otherwise) and preset detection math
  if extracted.
- App builds for macOS AND iOS destinations (compile gate).
- Manual: gallery click → backdrop appears + frame snaps to Square; second
  backdrop does not re-snap; explicit frame choice never overridden;
  Square export produces 1080×1080 mp4 (via app or `banny ship` on the
  saved project).
